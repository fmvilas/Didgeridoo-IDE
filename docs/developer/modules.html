Modules

This page contains information about how to manage modules with Didgeridoo as well as how to define them.

What is a module

A module is a fully -or almost fully- functional program that not belongs to the Didgeridoo core but enhances the functionality of the system. Modules differs from Libraries in a few things:

  • A module is not a collection but an specific enhancement.
  • Communication between the Core and the Modules, and between Modules, are not directly made but through the Facade. It provides a loose coupled system.
  • Modules usually will make use of Libraries.
