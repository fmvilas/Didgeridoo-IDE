Libraries

This page contains information about how to manage libraries with Didgeridoo as well as how to define them.

What is a library

Say library, say framework. We understand libraries as a collection of useful routines to help developer make things easier, faster and better.

On Didgeridoo, libraries stands for enhancing. We are using jQuery as our default framework but not everybody uses the same libraries so you will want to include new libraries. Thus, we provide an easy way to incorporate them to the Didgeridoo ecosystem.

Why to separate

It's easy. Libraries are not Didgeridoo, it makes use of them. By separating, the users can use many different libraries, multiple versions of the same library and use the ones they like for their projects without conditioning them to choose between a few.

How to define

IMPORTANT: This is not the way the developer should add libraries, it is the description of how it should be done.

Telling Didgeridoo how to include a library is easy. You only have to create a new folder inside the libraries folder with the name of the library, copy the files there and add the library to the libraries.list.json. That's all.

After this you will be able to load the library.
